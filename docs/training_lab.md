# Tarok AI Training Lab Architecture

The Tarok AI Training Lab implements a sophisticated reinforcement learning pipeline specifically designed for the complexities of Slovenian Tarok. Given the massive imperfect-information state space, structured bidding mechanics, and multi-phase game structure, we utilize a multi-pronged approach that bridges behavioral cloning and cutting-edge reinforcement learning paradigms.

Below is a detailed breakdown of the current approaches and activated features within our training lab.

## 1. Multi-Phase Training Pipeline

We employ a structured three-phase pipeline (`run_pipeline_cli`) to transition the agent from basic rule comprehension to advanced strategic execution.

### Phase 1: Imitation Pre-training (Instructed Learning)
Before exposing the agent to the high-variance environment of raw reinforcement learning, we bootstrap its policy networks using supervised learning on expert heuristics.
*   **Behavioral Cloning:** The agent learns by mimicking rollouts generated by our embedded algorithmic bots (`StockŠkis`).
*   **Purpose:** Rapidly encodes the foundational rules, basic card evaluation hierarchies, and sensible baseline strategies for bidding and playing without suffering from the "cold start" problem in sparse-reward environments.

### Phase 2: PPO Fine-tuning against Heuristic Bots
Once the agent understands the basics, it uses Proximal Policy Optimization (PPO) to explore and optimize its strategies against the established heuristic bots (e.g., StockŠkis v1-v3).
*   **Plateau Monitoring:** Training continues dynamically until win-rate improvement plateaus against the heuristic baselines, ensuring maximum sample efficiency.

### Phase 3: Fictitious Self-Play (FSP)
To achieve superhuman capability and prevent catastrophic forgetting, the agent shifts from playing static bots to playing against historical versions of itself.
*   **Mechanism:** The agent's opponents are sampled from a pool of its own frozen past checkpoints.
*   **Purpose:** Ensures the agent learns robust, unexploitable strategies and can adapt to diverse meta-game shifts, rather than overfitting to the fixed behaviors of the StockŠkis bots.

## 2. Advanced Algorithmic Features & Paradigms

Several key research features are activated during the reinforcement learning phase:

### Perfect Training, Imperfect Execution (PTIE) / Oracle Critic
*   **Concept:** In actor-critic frameworks like PPO, the "Critic" evaluates the current state to reduce variance, while the "Actor" chooses the action.
*   **Implementation:** We activate an **Oracle Critic** (`--oracle`). During training, the critic network is fed *perfect information* (visibility of all opponents' hidden cards and the undealt talon), allowing for highly accurate value estimations. The Actor network, which is structurally deployed for actual gameplay, still operates strictly on the *imperfect information* available to a real player.

### Session-Based Score Maximization
*   **Concept:** Standard trick-taking RL often optimizes for single-game win rates. However, Tarok is a series-driven game where relative point differentials matter immensely.
*   **Implementation:** We implement *session-based training* where an episode consists of multiple games (e.g., 20 games per session). The agent learns to maximize cumulative scores across the entire session. This naturally induces advanced strategic behaviors, such as:
    *   Knowing when to defensively pass and force a *Klop*.
    *   Recognizing when to bid conservatively (Three/Two) vs. aggressively (Solo).
    *   Risk management over a multi-game horizon rather than greedy single-game play.

### Multi-Decision Heads (`tarok.adapters.ai.network`)
*   Tarok is not a single continuous action space. We use an architecture with a shared neural trunk and multiple specialized output heads for discrete phases:
    1.  Bidding
    2.  King Calling
    3.  Talon Selection
    4.  Announcements
    5.  Card Play
*   This allows the agent to build unified latent representations of the game state while executing highly specific policies corresponding to the current game phase.

## 3. Evolutionary and Population-Based Training

To scale hyperparameter tuning and model robustness, the lab includes an evolutionary strategy component (`run_evolution` and `run_breeding_cli`).
*   **Evolutionary Optimizer / Breeding:** Instead of training a single agent natively, we maintain a population of distinct agents. These agents mutate hyperparameters, compete against one another, and breed the most successful traits to future generations. This ensures optimal learning rates, PPO clipping parameters, and FSP ratios without manual grid-searching.

## 4. High-Throughput Engine Integration
All RL pipelines suffer from the execution bottleneck of the environment simulator.
*   **Rust Game Engine:** The Python training loop interfaces seamlessly with a hyper-optimized Rust-based Tarok engine (`tarok_engine`).
*   **Impact:** Enables the generation of thousands of games per second, massively accelerating rollout collection for PPO and Imitation Learning. This high-throughput capability is what makes session-based and evolutionary approaches computationally feasible.
